Java Versions & Features

An interview-focused guide to the important features available in Java 8, 11, 17, 21, and 25.

Java ships a feature release every six months. These versions are especially important because most vendors treat them as Long-Term Support (LTS) releases.

Quick comparison

Version Released Main developer-facing highlights
Java 8 March 2014 Lambdas, streams, functional interfaces, default methods, Optional, Date/Time API
Java 11 September 2018 Standard HTTP Client, single-file programs, lambda parameter var, new String and Files methods
Java 17 September 2021 Sealed classes, stronger JDK encapsulation, enhanced random generators; includes records, text blocks, and pattern matching for instanceof
Java 21 September 2023 Virtual threads, record patterns, pattern matching for switch, sequenced collections, generational ZGC
Java 25 September 2025 Compact source files, flexible constructor bodies, module imports, scoped values, compact object headers

Java 8

Java 8 introduced functional-style programming to mainstream Java and remains a common interview baseline.

Lambda expressions and functional interfaces

List<String> names = Arrays.asList("Asha", "Mohit", "Ravi");
names.forEach(name -> System.out.println(name));
  • A lambda supplies an implementation for a functional interface.
  • A functional interface has exactly one abstract method.
  • Common interfaces include Predicate, Function, Consumer, and Supplier.

Streams API

List<String> longNames = names.stream()
    .filter(name -> name.length() > 4)
    .map(String::toUpperCase)
    .collect(Collectors.toList());

Streams describe a pipeline of operations. Intermediate operations such as filter and map are lazy; a terminal operation such as collect or forEach starts processing.

Other important additions

  • Method references, such as System.out::println.
  • Default and static methods in interfaces.
  • Optional for explicitly representing a possibly absent value.
  • The modern java.time Date and Time API.
  • CompletableFuture and concurrency improvements.
  • Metaspace replaced the Permanent Generation.

Java 11

Java 11 was the first widely adopted LTS release after Java 8. It also reflects major platform changes introduced in Java 9 and 10, including modules and local variable type inference.

Features introduced in Java 11

  • The HTTP Client became a standard API with synchronous and asynchronous requests.
  • Run a single source file directly with java Hello.java.
  • Use var in lambda parameters, mainly to allow annotations.
  • Java Flight Recorder became available as an open-source profiling tool.
  • TLS 1.3 support and the experimental Z Garbage Collector were added.
  • Java EE and CORBA modules were removed from the JDK.

Convenient APIs

"  ".isBlank();
"one\ntwo".lines();
"ha".repeat(3);
" text ".strip();

String content = Files.readString(path);
Files.writeString(path, content);

Java 17

Java 17 completed several language features developed between Java 11 and 17. It is a popular modern baseline for production applications.

Features introduced in Java 17

  • Sealed classes restrict which classes may extend or implement a type.
  • Enhanced pseudo-random number generator APIs.
  • Strong encapsulation of JDK internals.
  • Pattern matching for switch entered preview.
sealed interface Shape permits Circle, Rectangle {}

final class Circle implements Shape {}
final class Rectangle implements Shape {}

Important features available by Java 17

  • Records for concise immutable data carriers, finalized in Java 16.
  • Pattern matching for instanceof, finalized in Java 16.
  • Text blocks for multiline strings, finalized in Java 15.
  • Helpful NullPointerException messages.
  • Switch expressions.
record User(long id, String name) {}

if (value instanceof String text) {
    System.out.println(text.toUpperCase());
}

Java 21

Java 21's headline feature is virtual threads, which make thread-per-request server code practical at much larger scales.

Virtual threads

try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> callRemoteService());
}

Virtual threads are lightweight threads managed by the JVM. They improve scalability for workloads that spend significant time blocked on I/O. They do not make CPU-bound work inherently faster.

Other important additions

  • Pattern matching for switch became final.
  • Record patterns enable destructuring record values.
  • Sequenced collections provide consistent first, last, and reversed operations.
  • Generational ZGC improves low-latency garbage collection.
  • String templates, scoped values, and structured concurrency were preview features.
static String describe(Object value) {
    return switch (value) {
        case Integer number -> "integer: " + number;
        case String text    -> "text: " + text;
        case null           -> "null";
        default             -> "other";
    };
}

Java 25

Java 25 reached general availability on September 16, 2025. It continues work on simpler Java programs, performance, observability, and structured concurrency.

Language and library additions

  • Compact source files and instance main methods simplify small programs.
  • Flexible constructor bodies allow validation and preparation before calling super(...).
  • Module import declarations make all exported packages of a module available with one declaration.
  • Scoped values became final for safely sharing immutable data with callees and child threads.
  • A Key Derivation Function API was added.
void main() {
    IO.println("Hello, Java 25!");
}

JVM and tooling additions

  • Compact object headers reduce object memory overhead.
  • Generational Shenandoah provides another low-pause garbage collection option.
  • Ahead-of-time method profiling and command-line ergonomics improve startup work.
  • Java Flight Recorder gained cooperative sampling, method timing, tracing, and experimental CPU-time profiling.
  • Structured concurrency remained a preview feature.

Interview takeaways

  • Java 8: know lambdas, streams, functional interfaces, Optional, and java.time.
  • Java 11: know the HTTP Client, single-file execution, useful String methods, and migration removals.
  • Java 17: know records, sealed classes, pattern matching, text blocks, and stronger encapsulation.
  • Java 21: understand virtual threads and pattern matching for switch.
  • Java 25: recognize simplified source files, scoped values, and the latest JVM improvements.
